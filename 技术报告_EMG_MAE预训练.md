## 1. 项目概述

### 1.1 什么是EMG

**EMG (Electromyography，肌电图)** 是记录肌肉电活动的技术。我们的数据集包含：
- **64通道**表面EMG传感器
- **采样率**: 2000 Hz（每秒2000个数据点）
- **原始数据量**: 786,527个时间点 × 64个通道
- **动作类型**: 16类不同手势动作

### 1.2 什么是预训练

预训练是深度学习中的一种**迁移学习**技术：

```
传统方法：随机初始化 → 直接训练下游任务（容易过拟合，泛化能力差）

预训练方法：
1. 在大型无标注数据集上用MAE进行"自监督学习"
2. 学习通用的特征表示
3. 将学到的特征迁移到下游任务（如动作分类）
```

### 1.3 为什么使用MAE

**MAE (Masked Autoencoder)** 是一种自监督学习算法：

- 核心思想：随机遮挡输入图像的部分区域，让模型学习重建被遮挡的部分
- 优势：无需人工标注标签，只需图像本身
- 类比：类似于让学生做完形填空来理解文章大意

## 2. 数据处理流程

### 2.1 流程总览

```
原始EMG数据 (786527, 64)
    ↓
[1] 带通滤波 (1-100Hz)
    ↓
滤波后数据
    ↓
[2] RMS计算 (窗口64, 步长10)
    ↓
RMS特征 (~78593, 64)
    ↓
[3] 滑动窗口切分 (窗口64, 步长10)
    ↓
时间窗口 (~7853, 64, 64)
    ↓
[4] 归一化 + Jet Colormap
    ↓
最终图像 (7853, 3, 64, 64)
```

### 2.2 每一步详解

#### 步骤1：带通滤波 (Bandpass Filtering)

**目的**: 去除噪声，保留有意义的EMG信号

**参数**:
- 通带: 1-100 Hz
- 阶数: 4阶 Butterworth滤波器
- 采样率: 2000 Hz

**原理**:
```python
# 使用scipy实现零相位滤波
b, a = butter(4, [0.0005, 0.05], btype='band')  # 归一化频率
filtered = filtfilt(b, a, data)  # 零相位滤波，避免时间偏移
```

**为什么用Butterworth**: 通带内最平坦，滤波效果平滑

**为什么用filtfilt（零相位滤波）**: 普通滤波会造成时间偏移，filtfilt通过双向滤波消除偏移

#### 步骤2：RMS计算 (Root Mean Square)

**目的**: 将原始信号转换为包络特征，提取信号能量

**参数**:
- 窗口大小: 64样本（约32ms）
- 步长: 10样本（约5ms）

**原理**:
```
RMS = sqrt(1/N * Σ(xi^2))

对于每个通道的每个滑动窗口：
- 计算窗口内所有样本的平方和
- 除以窗口长度取平均
- 开方得到RMS值
```

**物理意义**: RMS反映了肌肉收缩的平均力量/能量

**为什么窗口64**: 32ms窗口能捕捉到肌肉收缩的基本特征，又不会包含太多变化

#### 步骤3：滑动窗口切分

**目的**: 将连续的RMS特征序列转换为独立的样本

**参数**:
- 窗口大小: 64帧
- 步长: 10帧

**输出形状**: (n_windows, 64, 64) → 时间 × 通道

**处理方式**: 每个64帧的时间窗口，转置为64×64矩阵（时间×通道）

#### 步骤4：归一化与Colormap转换

**目的**: 将数值矩阵转换为可视化RGB图像

**归一化**:
```python
normalized = (window - min) / (max - min)  # Min-Max归一化到[0,1]
```

**Jet Colormap映射**:
```python
colored = matplotlib.cm.jet(normalized)  # 数值映射到颜色
rgb = colored[:, :, :3]  # 提取RGB三个通道
```

**颜色含义**:
- 蓝色(低) → 肌肉活动弱
- 绿色/黄色(中) → 肌肉活动中等
- 红色(高) → 肌肉活动强

## 3. MAE模型架构

### 3.1 整体结构

```
输入图像 (3, 64, 64)
    ↓
[1] Patch Embedding (8×8 patches)
    ↓
64个patch tokens (64, 128)
    ↓
[2] 随机Mask (75% patches被遮挡)
    ↓
可见patches (16个) → Encoder → Encoder输出
    ↓                              ↑
Mask patches (48个) → 可学习Mask Token
    ↓
[3] Decoder (重建被遮挡的patches)
    ↓
预测的Mask patches (48, 192)
    ↓
与真实Mask patches计算MSE Loss
```

### 3.2 核心组件

#### Patch Embedding

**作用**: 将图像切分成patch，并线性投影到嵌入空间

```python
# 64×64图像，8×8 patch大小 → 8×8 = 64个patches
# 每个patch: 8×8×3 = 192维
# 投影后: 128维

self.proj = nn.Linear(8*8*3, 128)  # 线性投影层
```

#### 位置编码 (Position Embedding)

**作用**: 为每个patch添加位置信息，使模型知道patch的空间位置

```python
self.position_embedding = nn.Parameter(torch.randn(1, 64, 128))
# 可学习参数，64个位置，每个128维
```

**为什么需要**: Transformer本身不感知位置信息，需要显式添加

#### Mask Token

**作用**: 替代被遮挡patch的可学习占位符

```python
self.mask_token = nn.Parameter(torch.randn(1, 1, 128))
# 在encoder中看不到被遮挡的patch，用这个token代替
```

#### Encoder (Transformer Encoder)

**作用**: 处理可见的patch，提取高级特征

```python
encoder_layer = nn.TransformerEncoderLayer(d_model=128, nhead=4, dim_feedforward=512)
encoder = nn.TransformerEncoder(encoder_layer, num_layers=6)
# 6层，每层4头注意力，512维前馈网络
```

**工作流程**:
1. 16个可见patch token + 位置编码
2. 通过6层Transformer编码
3. 输出16个编码后的token

#### Decoder (Transformer Decoder)

**作用**: 根据可见token重建被遮挡的patch

```python
decoder_layer = nn.TransformerDecoderLayer(d_model=64, nhead=2, dim_feedforward=256)
decoder = nn.TransformerDecoder(decoder_layer, num_layers=2)
# 2层，2头注意力
```

**特殊设计**:
- 使用64维（比encoder的128维小）
- 接收所有64个token（可见+mask），但只看可见token的信息
- 输出所有64个位置的预测

### 3.3 Mask策略

```python
def get_random_indices(batch_size):
    # 生成随机排列的索引
    rand_indices = torch.argsort(torch.rand(batch_size, 64), dim=-1)
    # 前48个被mask，后16个可见
    mask_indices = rand_indices[:, :48]   # 75%被遮挡
    unmask_indices = rand_indices[:, 48:]  # 25%可见
    return mask_indices, unmask_indices
```

**为什么75%**: 高mask率迫使模型学习更好的特征表示（类似"做完形填空比选词填空学得更多"）

## 4. 训练流程

### 4.1 数据划分

```python
# 80%训练，20%验证
train_data, val_data = create_train_val_split(data_dir, val_ratio=0.2, seed=42)
# 7859样本 → 训练6287，验证1572
```

### 4.2 训练配置

| 参数 | 值 | 说明 |
|------|-----|------|
| Epochs | 300 | 完整数据集训练300轮 |
| Batch Size | 64 | 每批64个样本 |
| Learning Rate | 1e-4 | 初始学习率 |
| Weight Decay | 0.05 | 权重正则化 |
| Warmup Steps | 10% | 前10%步预热 |

### 4.3 学习率调度

**Warmup + Cosine Annealing**:

```
学习率
  ↑
 1e-4|─────────────\         (预热阶段线性增长)
     |            /
     |           /
     |          /
     |         /
     |        /
 1e-6|──────/                   (余弦退火衰减)
     └──────────────────────→ 步数
     0    10%    50%    100%
```

### 4.4 损失函数

```python
loss = MSE(predicted_masked_patches, true_masked_patches)
# 只在被遮挡的patches上计算损失
```

**为什么只用masked**: 可见部分的重建没有意义，模型已经"看到"了

### 4.5 保存策略

- **每20 epoch**: 保存检查点 `checkpoint_epoch_N.pth`
- **最佳模型**: 验证集损失最低时保存 `best_model.pth`
- **最终模型**: 训练结束保存 `final_model.pth`

## 5. 初学者指南：每一步的深度学习原理

### 5.1 为什么深度学习需要预处理？

**类比**: 想象你教小孩认识猫狗
- 原始EMG数据就像一团混乱的波形
- 预处理就像把关键特征提取出来，制成卡片

**原始信号的问题**:
- 包含太多噪声（环境电磁干扰、电极接触不良等）
- 数据量太大（786527个时间点太多）
- 数值范围不一致

### 5.2 什么是滤波？

**通俗理解**: 滤波就像筛子，只让符合条件的东西通过

**带通滤波**: 只保留1-100Hz的信号
- 低于1Hz的可能是电极漂移
- 高于100Hz的可能市电干扰

**代码解释**:
```python
b, a = butter(4, [0.0005, 0.05], btype='band')
# 创建了一个4阶带通滤波器
# [0.0005, 0.05]是归一化频率（相对于奈奎斯特频率）
```

### 5.3 什么是RMS？为什么用它？

**RMS (均方根)**: 就像求一段时间内的"平均强度"

```python
# 简单例子
signal = [3, 4, 3, 4]  # 模拟肌肉信号
rms = sqrt(mean(signal^2)) = sqrt((9+16+9+16)/4) = sqrt(12.5) ≈ 3.54
```

**为什么用RMS处理EMG**:
- EMG信号有正有负，直接平均会抵消
- RMS能反映真实的能量/强度

### 5.4 什么是滑动窗口？

**通俗解释**: 就像用放大镜在时间轴上移动，每次看一小段

```
时间轴: |----A----|----B----|----C----|----D----|...
        ↑窗口1            ↑窗口2            ↑窗口3
        窗口大小64        滑动步长10
```

### 5.5 什么是归一化？为什么需要？

**问题**: 假设两个通道的数值范围差很大
- 通道1: 0.001 ~ 0.01
- 通道2: 1000 ~ 10000

**归一化后**: 都变成0~1，模型能同等关注

```python
normalized = (x - min) / (max - min)
# 线性映射到[0,1]区间
```

### 5.6 什么是Colormap？

**通俗解释**: 给数值涂颜色，方便人眼识别模式

```
数值小 → 蓝色（冷色）→ 肌肉活动弱
数值中 → 绿色/黄色
数值大 → 红色（热色）→ 肌肉活动强
```

**为什么转成RGB**: 图像处理的标准格式是3通道RGB

### 5.7 什么是Patch？

**通俗解释**: 把大图切成小块的拼图

```
64×64图像，8×8 patch:
┌────┬────┬────┬────┬────┬────┬────┬────┐
│ P1 │ P2 │ P3 │ P4 │ P5 │ P6 │ P7 │ P8 │
├────┼────┼────┼────┼────┼────┼────┼────┤
│ P9 │...│    │    │    │    │    │P16 │
├────┼────┼────┼────┼────┼────┼────┼────┤
│... │    │    │    │    │    │    │    │
├────┼────┼────┼────┼────┼────┼────┼────┤
│P57 │    │    │    │    │    │    │P64 │
└────┴────┴────┴────┴────┴────┴────┴────┘
共64个patches，每个8×8像素
```

### 5.8 什么是Embedding？

**通俗解释**: 把复杂的东西变成数字向量，方便计算

```
"猫" → [0.2, -0.5, 0.8, ...]  (128维向量)
"狗" → [0.1, -0.3, 0.9, ...]  (128维向量)
```

**为什么需要**: 神经网络只懂数字，不懂"像素值"或"文字"

### 5.9 什么是位置编码？

**问题**: 把图像切成patches后，模型不知道哪个patches在哪里

**解决方案**: 给每个patch添加一个表示位置的向量

```python
# 就像给拼图块编号
patch_1_pos = [1, 0, 0, 0, ...]  # 左上角
patch_2_pos = [0, 1, 0, 0, ...]  # 上方
```

### 5.10 什么是注意力机制？

**通俗解释**: 让你"关注"重要的部分

```
看这张图时，你会先看整体还是局部？
注意力机制让模型自己学习关注哪里
```

**自注意力**: 每个位置和其他所有位置计算关系

```python
# 简化示意
attention = softmax(query @ key.T) @ value
# query: 我在看什么
# key: 我有关注什么
# value: 如果匹配，我应该提取什么信息
```

### 5.11 什么是Transformer？

**通俗解释**: 一种 powerful 的神经网络架构，核心是注意力机制

```
Transformer = 自注意力 + 前馈网络 + 残差连接

类比: Transformer像一个翻译团队
- 每个人都能看到其他所有人的工作（自注意力）
- 每个人负责自己的专业部分（前馈网络）
- 重要信息可以快速传递（残差连接）
```

### 5.12 什么是Encoder和Decoder？

**Encoder (编码器)**:
- 输入: 可见的patches
- 输出: 这些patches的"含义"（特征向量）

**Decoder (解码器)**:
- 输入: Encoder的输出 + mask tokens
- 输出: 重建被遮挡的部分

**类比**:
- Encoder像阅读理解，找出文章重点
- Decoder像完形填空，根据重点填空白

### 5.13 什么是Mask？

**通俗解释**: 遮挡输入的一部分，让模型猜被挡住的内容

```
原图:  [🏠🌳🚗👤🌺]
Mask:  [🏠[???][???][???][🌺]
       可见部分    被遮挡
```

**为什么随机**: 避免模型记住位置，学习真正的特征

### 5.14 什么是损失函数？

**通俗解释**: 衡量"猜错了多少"

```python
loss = MSE(pred, target)
# MSE = mean((预测值 - 真实值)²)
# loss越小，预测越准
```

### 5.15 什么是学习率？

**通俗解释**: 每次"猜"的时候迈多大步子

```
学习率太大: 步子太大，容易跳过正确答案
学习率太小: 步子太小，学得太慢
合适的学习率: 稳步前进，最终收敛
```

### 5.16 什么是Epoch？

**通俗解释**: 把所有样本"看"一遍

```
一个epoch = 喂给模型所有6287个训练样本一次

300 epochs = 把所有样本看了300遍
```

### 5.17 什么是Batch Size？

**通俗解释**: 一次喂多少样本

```
Batch size 64 = 一次喂64张图给模型
               ↓
              模型同时学习这64张图的特点
               ↓
              更新一次参数
```

**为什么不能一次喂完所有**: 显存不够，且泛化能力可能变差

### 5.18 什么是验证集？

**通俗解释**: 用模型没见过的题来测试

```
训练集: 课本习题 → 边学边做，老师批改
验证集: 模拟考试 → 检验学习效果，不告诉答案
测试集: 正式考试 → 最终能力评估
```

**为什么需要**: 避免"死记硬背"，检验真实泛化能力
